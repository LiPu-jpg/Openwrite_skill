# 后置验证系统

零 LLM 成本的确定性规则检测，快速检查章节质量。

## 核心概念

**验证严重性**：
- `error` - 必须修复（禁止句式等）
- `warning` - 建议修复（AI味信号等）

## 检测规则

### 禁止句式（error）
| 规则 | 说明 |
|------|------|
| 不是...而是... | 强制转折句式 |
| —— | 破折号 |
| ··· 或 。。 | 省略号 |

### 元叙事标记（warning）
| 标记 | 说明 |
|------|------|
| "到这里算是" | 作者旁白 |
| "读者可能" | 作者旁白 |
| "作者想说" | 作者旁白 |

### 报告术语（error）
| 术语 | 说明 |
|------|------|
| 核心动机 | 策划案术语 |
| 信息边界 | 策划案术语 |
| 叙事支撑 | 策划案术语 |
| 弧线转折 | 策划案术语 |

### 高疲劳词（warning）
| 词 | 建议频率 |
|----|---------|
| 突然 | ≤1次/章 |
| 瞬间 | ≤1次/章 |
| 旋即 | ≤1次/章 |
| 骤然 | ≤1次/章 |

### 转折词密度（warning）
- 超过 1次/3000字 触发警告

## 可用工具

| 工具 | 说明 |
|------|------|
| `validate_post_write` | 后置规则验证 |

## 使用示例

### 检查最新章节

```
用户: 检查一下最新章节

AI:
[COMMAND] validate_post_write {
  "chapter_id": "latest"
}

结果:
✅ 通过: 否
❌ 错误: 2个
  - 禁止句式：'不是...而是...'
  - 报告术语：'核心动机'
⚠️  警告: 3个
  - 高疲劳词：'突然' 出现 3 次
  - 转折词密度过高
  - 元叙事标记：'读者可能'
```

### 检查指定章节

```
用户: 第5章有没有问题？

AI:
[COMMAND] validate_post_write {
  "chapter_id": "ch_005"
}

结果:
✅ 通过: 是
❌ 错误: 0个
⚠️  警告: 0个

章节质量检查通过！
```

## 修复建议

### 禁止句式修复

**问题**: "他不是普通人，而是天才"

**修复**: "他是天才，不是普通人"（改为平铺直叙）

### 高疲劳词修复

**问题**: "他突然笑了"、"突然转身"

**修复**: 替换为更精确的动作描写

### 元叙事修复

**问题**: "读者可能觉得主角太弱了"

**修复**: 删除或改为角色内心独白

## 使用时机

1. 写完章节后立即检查
2. 提交前最终检查
3. 发现逻辑问题时排查
