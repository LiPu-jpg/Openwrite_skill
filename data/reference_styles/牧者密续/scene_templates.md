# 场景模板 (Scene Templates)

> 记录《牧者密续》的核心场景类型、结构模板与写法要点。

---

## 模板1：调查推理场景

> 哈伊娜或艾华斯对案件/事件进行现场调查的场景。

### 结构
```
到达现场 → 环境描写(维多利亚街景/建筑) → 发现线索 → 分析推理
→ 遇到阻碍/新角色 → 获取关键信息 → 形成假说 → 悬念收尾
```

### 写法要点
- **环境先行**：抵达时先描写场景氛围（雾气、煤气灯、建筑格局）
- **线索可视化**：线索要让读者"看到"才有参与感
- **双轨推理**：
  - 哈伊娜视角：按监察局训练方法推理（正统）
  - 艾华斯视角：用游戏攻略知识定位（开挂）
- **权限受限**：哈伊娜常面临"权限不足"的调查阻碍
- **对话获取信息**：询问目击者、调取档案等

### 典型段落节奏
> 哈伊娜推开那扇陈旧的木门。
> 屋内弥漫着一股奇怪的甜腥气味。地板上是交叉排列的蜡烛残迹——不，那不是蜡烛。
> 她蹲下身，借着从百叶窗缝隙中透入的灰白光线仔细辨认。
> 「……是骨头。」

## 模板2：政治博弈场景

> 多方势力在场的会议、宴会或谈判场景。

### 结构
```
多方入场(身份/立场介绍) → 表面寒暄(暗藏试探) → 议题展开(多方发言)
→ 核心冲突暴露 → 话术交锋(层层递进) → 暂时平衡/暗流涌动 → 格局变化
```

### 写法要点
- **人物清单**：开场明确在场人物和各自阵营
- **多线并行**：至少3方势力的利益交织
- **话术描写**：每句对话标注说话者的"真实意图"（通过视角角色的分析）
- **艾华斯的优势**：作为"玩过游戏的人"，能看穿NPC们的立场
- **伊莎贝尔的成长**：从旁听者到参与者到决策者
- **奥斯瓦尔德的角色**：精灵管家在政治场景中是隐性实力展示

### 典型张力来源
- 谁在撒谎？（读者跟随主角分析）
- 谁知道艾华斯的秘密？（穿越者身份的暴露风险）
- 这场谈判的实际输赢如何？（表面妥协 vs 实际得利）

## 模板3：战斗+游戏机制场景

> 结合超凡能力和游戏系统的战斗场景。

### 结构
```
敌人登场(能级/道途判断) → 艾华斯游戏分析(弱点/攻略回忆) → 战前布置
→ 战斗爆发(技能描写+伤害交换) → 转折(新敌/新技/队友介入) → 胜利/撤退
→ 战后清算(经验/伤势/情报)
```

### 写法要点
- **游戏框架覆盖**：战前分析用游戏术语（"这个BOSS在游戏里是三阶段战"）
- **技能可视化**：每个技能的视觉效果要详细描写
  > 黑色的火焰从他的掌心蔓延而出——不是物理意义上的火焰，而是恶魔之力扭曲了光线本身。
- **伤害有代价**：所有战斗都有明确代价（法力耗尽、身体损伤、精神污染）
- **队友协同**：多人战斗中强调配合与分工
- **游戏与现实的差异**：「在游戏里这个BOSS只是二阶BOSS……但现在他分明展现出了三阶的力量」→紧张感来源

### 动作描写特征
- 中-高速节奏（不像天启预报那幺暴烈极端）
- 穿插分析段落（"他注意到对方的左手在发抖"）
- 战斗中仍有对话（一边打一边交流是常态）
- 受伤描写克制但具体（不过度血腥）

## 模板4：社交/日常温馨场景

> 角色间非任务状态的日常互动。

### 结构
```
场景设定(客厅/餐厅/花园) → 角色聚集 → 轻松对话/吐槽 → 不经意间的情感流露
→ 某个温暖的瞬间定格 → 平静收尾(或被突发事件打断)
```

### 写法要点
- **食物描写**：频繁出现（茶、点心、早餐是维多利亚日常的锚点）
  > 奥斯瓦尔德端来了新沏的红茶和一盘刚出炉的司康饼。
- **角色互动的层次**：
  - 艾华斯↔奥斯瓦尔德：主仆但更像老友，互相讽刺
  - 艾华斯↔伊莎贝尔：微妙暧昧 + 政治同盟
  - 艾华斯↔夏洛克：默契的信任
  - 哈伊娜↔任何人：社恐式紧张感
  - 莉莉：影子般的存在感，但递茶递东西 = 情感表达
- **核心功能**：让读者"住进"莫里亚蒂宅邸，产生归属感
- **被打断的日常**：温馨场景常被突发事件打断 → 反差制造紧迫感

## 模板5：导师讲授/信息揭示场景

> 通过知情角色向主角传递世界观知识。

### 结构
```
问题提出(主角遇到理解瓶颈) → 导师登场(马瑟斯/米娜/精灵长辈等)
→ 渐进讲述(从现象到本质) → 关键概念(术语正式定义)
→ 主角映射(与游戏知识对号) → 新的未知浮现
```

### 写法要点
- **不做纯倾泻**：讲述中穿插主角的反应、回忆、吐槽
- **双线理解**：导师讲"这个世界的规则" + 艾华斯映射到"游戏里的设定"
- **情感加载**：知识绑定感情故事（"当年那位大术师……其实是我的朋友"）
- **读者友好**：通过角色之口自然解说，避免说明文
- **每次只解一层**：不一次讲完，留悬念给后续章节

### 典型节奏
> 「你知道『衔尾之环』意味着什幺吗？」马瑟斯看向窗外的雨。
> 艾华斯心中一震。在游戏里，衔尾之环是最终副本的名字——但他从未想过它在"现实"中到底是什幺。
> 「……不太清楚。」他决定装作不知道。
> 「那是——」

## 模板6：穿越者内心推演场景

> 艾华斯独处时的游戏知识分析和策略规划。

### 结构
```
触发(获得新信息/遇到游戏中的事件) → 游戏回忆(该事件在游戏中的走向)
→ 偏差分析("这一世"已经不同了) → 策略推演(该怎幺做)
→ 风险评估(暴露穿越者身份的风险) → 决定行动方案
```

### 写法要点
- **双重叙事**：同一事件交替展示"游戏中的版本"和"现实版本"
- **游戏术语密集**：这类场景是术语最集中的地方
- **紧张感来源**：「如果不提前干预，三年后这里就是一片废墟」
- **蝴蝶效应焦虑**：「自己的存在已经改变了太多东西了」
- **孤独感**：这些推演只能一个人做，无法与任何人分享

## 模板7：多国外交/跨文化交流场景

> 不同国家角色的正式或半正式互动。

### 结构
```
国际背景简述 → 各国代表入场(服饰/礼节差异)
→ 多语言环境展示 → 利益博弈(每国一条线)
→ 文化冲突/误解 → 某方主导局面 → 新的国际格局
```

### 写法要点
- **语言差异**：明确标注说的什幺语（阿瓦隆语/星锑语/鸢尾花语等）
- **礼节差异**：各国行礼方式、称谓、座次不同
- **服饰区分**：通过服装描写强化国别感
- **多线利益**：至少呈现3国以上的不同诉求
- **艾玛的多语种试探**：经典外交手段，用不同语言的切换来试探对方的身份
- **信息层级**：
  - 读者看到：各方的表面发言
  - 主角分析：各方的真实意图
  - 作者暗示：未来的走向

## 模板8：进阶/升级仪式场景

> 超凡者进行能级提升的正式仪式。

### 结构
```
仪式准备(材料/条件/风险) → 仪式环境(圣殿/秘室/特殊空间)
→ 仪式过程(视觉化描写) → 内在体验(精神世界) → 突发状况/选择
→ 成功/代价 → 新能力展示
```

### 写法要点
- **游戏 vs 现实**：游戏中点两下按钮就进阶 → 现实中是生死考验
- **感官描写**：仪式中的超自然感受（灵魂被拉扯、看到异象等）
- **伏笔承兑**：进阶时常揭示之前铺垫的秘密
- **代价明确**：每次进阶都有不可逆的代价或改变

## 模板9：身份伪装/揭穿场景

> 角色伪装身份或被揭穿的场景。

### 结构
```
表象(角色以假身份行动) → 维持(应对各种试探)
→ 危机(某个细节暴露了) → 紧张对峙 → 化解/暴露
→ 后果(关系变化/新的信任/新的敌意)
```

### 写法要点
- **多层身份**：艾华斯的身份伪装是全书核心（穿越者装NPC、恶魔学者装善人、贵族装普通人）
- **读者的紧张**：读者知道真相但对方不知道 → 悬念
- **微表情/细节**：揭穿往往来自极小的细节
- **化解的机智**：艾华斯用话术巧妙绕过危机
- **被信任的角色知道一部分**：奥斯瓦尔德、夏洛克等知道"一些"但不知道"全部"

## 模板10：章末作者注场景

> 虽然不是故事场景，但已构成本作的标志性"场景"。

### 结构
```
正文结束(通常是悬念/温情收尾) → 分隔线
→ 作者按(1-5段) → 内容：字数汇报/日常/互动/推书/伏笔暗示
```

### 写法要点
- **不破坏正文情绪**：分隔线之后完全切换语境
- **口语化**：与正文的文学叙述形成鲜明反差
  > 「六千多字，够意思了吧哈哈」
  > 「评论区有人猜到了，厉害」
- **功能性**：
  - 更新频率管理（请假/加更通知）
  - 读者期待管理（预告接下来的内容方向）
  - 社区维护（回应评论、推荐其他作品）
- **偶尔的伏笔**：「注意看这章某个人说的某句话」→ 高级读者奖励

## 场景组合规律

### 典型章节内场景搭配
| 章节类型 | 场景组合 |
|----------|----------|
| 推进章 | 日常(20%) + 调查/政治(60%) + 悬念(20%) |
| 高潮章 | 战斗(60%) + 揭秘(20%) + 余波(20%) |
| 过渡章 | 日常温馨(50%) + 信息揭示(40%) + 铺垫(10%) |
| 多线章 | POV-A(40%) + POV-B(40%) + 交汇(20%) |

### 场景转换手法
- **视角切换**：直接跳到另一角色（最常用）
- **时间跳转**：「三天后——」「那天晚上——」
- **空间跳转**：场景描写开头标明新地点
- **信息串联**：上一场景的未解之谜在下一场景得到部分解答
