# 资源账本

## resources
| 资源 | 当前值 | 说明 |
|------|--------|------|
| 腕部符文 | 已觉醒 | 能稳定作为异常活跃的外显标记 |
| 逻辑链感知 | 入门 | 能分辨局部因果结构，尤其适合代码与设备异常 |
| 术式推演 | 初学 | 可做短时模拟，但一复杂就头痛、走神 |
| 小补丁能力 | 已解锁 | 能处理轻量异常，尚无法长期维持 |
| 现实掩护 | 一般 | “加班太累”“线上事故”仍能暂时解释一部分异常 |

## constraints
| 限制 | 风险级别 | 说明 |
|------|----------|------|
| 算力过载 | 高 | 精神消耗快，推演时间一长就会明显失真 |
| 日志留痕 | 高 | 第六章后已经形成真实物证 |
| 赵磊怀疑 | 中 | 朋友视角最容易看出日常细节变化 |
| 林月观察 | 中高 | 她不会轻易放过不合逻辑的地方 |
| 工作节奏 | 高 | 版本周期本身就在不断压缩主角的试错空间 |

## balances
- 能力越熟，工作效率越高；效率越高，越容易被问“你最近为什么这么稳”。
- 社会资源目前主要来自团队日常信任，还没有真正可依赖的“异常同盟”。
- 任何一次补丁成功都会带来短期安全感，但同时也会制造新的技术债与物证。
